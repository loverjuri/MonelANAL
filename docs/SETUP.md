# Инструкция по настройке модуля учёта ЗП и финансов

## Быстрый старт (уже получили ЗП и хотите начать сейчас)

Если вы уже получили зарплату (например, 10-го) и хотите сразу вести учёт с нового периода:

1. **Создайте таблицу** и выполните инициализацию (раздел 1).
2. **Deploy скрипта** (разделы 2–4): Apps Script, BOT_TOKEN, ChatID, webhook.
3. **Установите триггеры** — `installTriggers()` (раздел 5).
4. Готово. Бот сам определит, что последняя выплата была 10-го: период учёта — с 11-го до следующих 25-го. Начните отвечать на вопросы бота (18:00 — основная работа, 00:05 — вторая работа), 25-го придёт запрос о сумме выплаты.
5. Чтобы **вручную записать прошлые дни** (11, 12 марта и т.д.): в редакторе Apps Script выполните один раз, подставив даты и часы:
   ```javascript
   function addPastDays() {
     addWorkLogEntry('2025-03-11', 'Main', 8, 'Work');
     addWorkLogEntry('2025-03-12', 'Main', 8, 'Work');
     // добавьте строки для других дней при необходимости
   }
   ```
   Запустите `addPastDays` из списка функций (Запуск → addPastDays).

## Работа с ботом

### Команды и кнопки

При `/start` или `/help` бот показывает **главное меню** с кнопками: Статус | Расход | Доход | Справка. Можно нажать кнопку вместо ввода команды.

| Действие | Описание |
|----------|----------|
| **Статус** (кнопка или `/status`) | Накоплено ЗП с последней выплаты, вторая работа за месяц, остаток бюджета, дата следующей выплаты |
| **Расход** (кнопка или `/expense`) | Запись расхода: сумма → категория (Еда, Транспорт, Жильё, Здоровье, Развлечения, Прочее) → комментарий |
| **Доход** (кнопка или `/income`) | Запись внезарплатного дохода (премии, фриланс, подарки): сумма → комментарий |
| **Справка** (кнопка или `/help`) | Описание кнопок и команд |

### Автоматические вопросы бота

- **18:00 MSK** — как прошёл рабочий день? (основная работа)
- **00:05 MSK** — доходы со второй работы за вчера?
- **10-е и 25-е число, 10:00 MSK** — день выплаты: сколько фактически пришло на карту?
- **19:00 MSK** — напоминание по основной работе
- **00:30 MSK** — напоминание по второй работе

Зарплата и вторая работа записываются через ответы на эти вопросы. Расходы и прочие доходы — вручную через команды `/expense` и `/income`.

## 1. Создание Google Таблицы

1. Создайте новую Google Таблицу (например, «Учёт ЗП и финансов»).
2. Создайте листы и задайте заголовки столбцов **строго в указанном порядке**.

### Лист `Config` (Настройки)

| Parameter   | Value   |
|------------|---------|
| FixedSalary | 100000 |
| SecondJobPercent | 10 |
| PayDay1 | 10 |
| PayDay2 | 25 |
| WorkHoursNorm | 8 |
| ChatID | *ваш Telegram Chat ID* |
| TimeZone | Europe/Moscow |

**Примечание:** `BotToken` хранится в **Свойствах скрипта** в редакторе Apps Script (**Расширения** → **Apps Script** → **Проект** → **Свойства проекта** → вкладка **Свойства скрипта**): ключ `BOT_TOKEN`, значение — токен от @BotFather. В таблице можно не указывать или оставить пустым.

### Лист `ProdCalendar`

Столбцы: `Date` | `IsWorking` | `WorkHoursNorm` | `MonthNormHours`  
Заполняется автоматически скриптом (API isdayoff.ru). Заголовки задать вручную.

### Лист `WorkLog`

Столбцы: `ID` | `Date` | `JobType` | `HoursWorked` | `HourRateSnapshot` | `Status` | `SickDayIndex` | `IsPaid` | `CreatedAt`

### Лист `Orders`

Столбцы: `OrderID` | `Date` | `Description` | `Amount` | `Status`

### Лист `Finance`

Столбцы: `ID` | `Date` | `Type` | `Amount` | `Category` | `Comment`

### Лист `Calculations`

Столбцы: `PeriodStart` | `PeriodEnd` | `AccruedSalary` | `ReceivedSalary` | `Difference`

### Лист `State`

Столбцы: `ChatID` | `Scenario` | `Step` | `Payload` | `UpdatedAt`  
Используется для хранения состояния диалога с ботом (FSM).

### Лист `Logs`

Столбцы: `Timestamp` | `Level` | `Message`

**Инициализация:** В редакторе Apps Script выполните один раз `initSheetIfNeeded()` (создаёт листы и заголовки), затем `initConfigDefaultsIfEmpty()` (добавляет строки Config по умолчанию, если лист пустой).

## 2. Google Apps Script

1. В таблице: **Расширения** → **Apps Script**.
2. Создайте проект и привяжите его к этой таблице.
3. Скопируйте код из папки `src/` в файлы проекта (Main.gs, Config.gs, Bot.gs и т.д.). Либо используйте [clasp](https://github.com/google/clasp) для деплоя из репозитория.
4. Сохраните проект.

### Свойства скрипта

В **Проект** → **Свойства проекта** → вкладка **Свойства скрипта** добавьте:

- **BOT_TOKEN** — токен бота, выданный @BotFather.
- При желании **CHAT_ID** — ваш Telegram Chat ID (если не указан в листе Config).

## 3. Telegram-бот

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. Команда `/newbot`, следуйте инструкциям, получите **Token**.
3. Узнайте свой **Chat ID**: напишите боту [@userinfobot](https://t.me/userinfobot) и скопируйте Id. Либо после первого сообщения боту можно посмотреть Chat ID в логах (если временно залогировать входящий update).
4. Вставьте Token в свойства скрипта (BOT_TOKEN), Chat ID — в лист Config (столбец Value для параметра ChatID) или в свойство CHAT_ID.

## 4. Развёртывание webhook

1. В редакторе Apps Script (откройте его из **вашей** таблицы: Расширения → Apps Script): **Развёртывание** → **Новое развёртывание** → тип **Веб-приложение**.
2. Описание: например, «Webhook».
3. **Запуск от имени:** себя.
4. **У кого есть доступ:** обязательно **Все** (Anyone). Если выбрано «Только я» или «Только пользователи с аккаунтом Google», при переходе по URL появится экран входа, и Telegram не сможет вызвать скрипт — бот будет молчать.
5. Нажмите **Развернуть**. Скопируйте URL (заканчивается на `/exec`). Если развёртывание уже есть — **Развёртывание** → **Управление развёртываниями** → у нужного развёртывания нажмите «Редактировать» (карандаш), проверьте «У кого есть доступ: Все», при необходимости создайте новую версию.
6. Проверка: откройте URL в браузере **в режиме инкогнито** (или выйдя из Google). Должно отображаться «OK», а не экран входа. Если виден «Sign in» — доступ настроен неверно.
7. В коде `setTelegramWebhook()` подставьте URL в `webhookUrl`, выполните функцию. В логах должно быть `{"ok":true}`.

### Вариант: polling вместо webhook

Если webhook даёт сбои (дубли сообщений, «самоспам»), можно перейти на polling. Выполните **один раз** функцию `switchToPolling()` — webhook будет удалён, добавлен триггер опроса каждую минуту. Сообщения обрабатываются только при опросе, дубли и ложные срабатывания исключены. Чтобы вернуться на webhook: `switchToWebhook()`, затем `setTelegramWebhook()`.

## 5. Триггеры по расписанию

В редакторе Apps Script откройте файл с функцией `installTriggers`, затем выполните её **один раз** из списка функций (Запуск → выбрать `installTriggers`). Она создаст все нужные триггеры:

- 18:00 MSK — вопрос о рабочем дне (основная работа).
- 00:05 MSK — вопрос о доходах со второй работы за вчера.
- 10:00 MSK в 10-е и 25-е число — день выплаты.
- 1-е число месяца — обновление производственного календаря.
- 19:00 MSK — напоминание по основной работе.
- 00:30 MSK — напоминание по второй работе.

Часовой пояс в триггерах задаётся как Europe/Moscow где возможно.

---

## 6. Первый запуск в новом месяце

При первом обращении к расчётам в новом месяце скрипт сам подтянет производственный календарь (isdayoff.ru) на текущий и следующий месяц. Также календарь обновляется 1-го числа по триггеру.

---

## 7. Изменение оклада и сброс настроек

- **Оклад (FixedSalary):** измените значение в листе Config в строке с параметром `FixedSalary`. Кэш обновится в течение нескольких минут или при следующем запросе.
- **Даты выплат (PayDay1, PayDay2):** измените в Config соответственно.
- **Сброс состояния бота:** очистите лист State (удалите строки с данными, оставив заголовок) — диалог начнётся заново.
- **Переустановка триггеров:** в редакторе GAS откройте **Триггеры** (часы слева), удалите старые, снова выполните `installTriggers()`.

## Бот не реагирует: что проверить

### Если в журнале выполнения нет новых записей при отправке сообщений боту

Значит webhook-запросы **не доходят** до скрипта. Telegram не вызывает ваш URL.

1. **Проверьте webhook:** Выполните `checkWebhookInfo()` — в логах будет текущий URL или «не установлен». Если не установлен или URL другой — webhook нужно настроить.
2. **Получите URL развёртывания:** Скопируйте URL из **Развёртывание → Управление развёртываниями** (кнопка копирования у развёртывания типа «Веб-приложение»). URL должен заканчиваться на `/exec`, **не** на `/dev` — тестовый `/dev` для webhook не подходит.
3. **Проверьте доступность:** Откройте URL в браузере — должно отобразиться «OK».
4. **Установите webhook:** В `setTelegramWebhook()` подставьте URL в переменную `webhookUrl`. Скопируйте его из **Развёртывание → Управление развёртываниями** (кнопка копирования у развёртывания). Не используйте URL из адресной строки редактора (там есть `/edit`) и без фрагментов `#...`. Формат: `https://script.google.com/macros/s/.../exec`. Выполните функцию — в логах должно быть `{"ok":true}`. Если ранее был установлен неверный URL, сначала выполните `deleteTelegramWebhook()`, затем `setTelegramWebhook()`.
5. **Напишите боту снова** — в журнале выполнения должна появиться запись «doPost: запрос получен».

### Webhook указывает на другой проект

Если `checkWebhookInfo()` показывает URL с идентификатором `.../s/ААААААА/...`, а при открытии Управление развёртываниями в вашем проекте виден другой `.../s/БББББББ/...` — это **разные проекты**. Сообщения боту приходят в проект А, а вы редактируете проект Б. Решение: открывайте Apps Script **из вашей таблицы** (Расширения → Apps Script). Скопируйте URL развёртывания **этого** проекта. Выполните `deleteTelegramWebhook()`, затем `setTelegramWebhook()` с правильным URL.

### Появился экран «Sign in» при открытии URL развёртывания

Webhook вызывает URL без авторизации Google. Если в настройках развёртывания стоит «Только я» или «Только пользователи с аккаунтом Google», Telegram получит страницу входа вместо ответа — бот молчит. **Решение:** Развёртывание → Управление развёртываниями → Редактировать (иконка карандаша) у развёртывания → **У кого есть доступ:** выберите **Все** → Сохранить новую версию.

### Бот отвечает «Chat ID не в списке разрешённых», хотя ChatID указан

Конфигурация кэшируется на 5 минут. После добавления или изменения ChatID выполните `invalidateConfigCache()` в редакторе Apps Script — кэш сбросится, новое значение подхватится сразу. Проверьте также, что в Config или свойствах скрипта нет лишних пробелов.

### Если логи есть, но бот не отвечает

- **Chat ID:** Узнайте свой Chat ID через [@userinfobot](https://t.me/userinfobot) и впишите в Config (ChatID) или свойства скрипта (CHAT_ID). При неверном ID бот пришлёт сообщение с вашим ID.
- **Ошибки:** Смотрите журнал выполнения и лист Logs. При ошибке бот отправит «Произошла ошибка».
